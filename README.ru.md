# Sorta

> Языки: [English](README.md) · **Русский** · [日本語](README.ja.md)

**Индексирует и раскладывает большую коллекцию фото/видео** (протестировано на
60+ ГБ, рассчитано на 300+ ГБ) в чистую структуру папок — по **городу/стране**,
**человеку** или **событию** — с безопасностью в приоритете: по умолчанию dry‑run,
журнал перемещений и откат одной командой.

Sorta работает **локально** (ML‑модели для лиц, сцен и текста считаются на вашей
машине), **никогда не изменяет оригиналы** и предлагает и **CLI**, и пошаговое
**локальное веб‑приложение**.

> ⚡ **Для полноценного использования** — распознавание людей, глубокий
> VLM‑ярус или большие коллекции — рекомендуется **NVIDIA GPU (CUDA 13) с
> ≥ 4 ГБ VRAM** (**≥ 8 ГБ** для VLM‑яруса). На CPU всё тоже работает, просто для
> этих задач заметно медленнее — см. [Системные требования](#системные-требования).

> 📖 **Руководство пользователя:** [English](docs/guide/user-guide.en.md) ·
> [Русский](docs/guide/user-guide.ru.md) · [日本語](docs/guide/user-guide.ja.md)

## Возможности

- **Сортировка по городам / людям / событиям** из единого индекса — смена режима
  не требует пересканирования.
- **Офлайн‑геолокация** (bundled GeoNames) с GPS + наследованием по сессиям;
  опционально — онлайн Nominatim/OSM.
- **Быстрый базовый прогон по умолчанию:** `sorta run` / кнопка **Обработать** в UI
  делают раскладку по городам + поиск дублей (index → geo → landmarks → junk →
  почти‑дубли). **Лица и события — opt‑in** (`--faces`/`--events`, либо
  соответствующие чекбоксы) — это самые долгие этапы, нужны не всем.
- **Лица и люди:** локальная детекция + кластеризация (insightface) после
  включения; называйте и сливайте кластеры, затем сортируйте или собирайте
  персональные альбомы.
- **События:** кластеризация по разрывам времени + городу с локализованными
  именами, после включения; плюс ручные события.
- **Дубликаты:** точные (blake3) и почти‑дубликаты (перцептивный хэш) с
  UI для пакетного разбора.
- **Мусор и документы:** скриншоты/мемы выводятся отдельно; документы
  собираются в review‑папку `_Документы/` (CLIP + плотность текста).
- **Альбомы:** собрать срез по человеку/событию в именованную папку через
  **жёсткие ссылки** (почти без доп. места), копию или перемещение.
- **Локальное веб‑приложение** (`sorta ui`): обработать папку, посмотреть план,
  разобрать дубли, назвать людей и применить сортировки/альбомы — всё в браузере.
  Вкладки **Люди**/**События** появляются только после того, как эти этапы
  реально прогнаны.
- **Трёхъязычность** UI и имён папок: **ru / en / ja**.
- **Безопасность по умолчанию:** dry‑run, журнал + `undo`, сверка blake3,
  никогда не перезаписывает (суффикс `_1`, `_2`).

## Системные требования

| | Профиль CPU (`--extra cpu`) | Профиль GPU (`--extra gpu`) |
|---|---|---|
| Железо | Любая машина x86‑64 | NVIDIA GPU + драйвер с поддержкой **CUDA 13** |
| VRAM | н/д | **~3 ГБ** база + лица (замерено на RTX 5090: CLIP ViT‑L 2.0 ГБ + buffalo_l 0.6 ГБ) — **≥ 4 ГБ** комфортно, **≥ 8 ГБ** для глубокого VLM‑яруса (Qwen2.5‑VL‑3B, ~7 ГБ оц.) |
| Скорость лиц/CLIP | Работает, но **медленно** (часы на большой коллекции с включёнными лицами/событиями) | Быстро — ориентир: 6298 фото, лица+события+junk ≈ **45 мин** (быстрый ярус) / ≈ **77 мин** с опциональным глубоким VLM‑ярусом |
| Подходит для | Раскладки по городам + дублей на любой машине; небольших коллекций с включёнными лицами/событиями | Больших коллекций (300 ГБ+) с регулярно включёнными лицами/событиями |

Общее для обоих профилей: Python **3.11–3.14**, [`uv`](https://docs.astral.sh/uv/),
и **`exiftool` на PATH** (обязателен для метаданных HEIC/RAW/видео — без него Sorta
откатывается на Pillow, которая читает только JPEG/PNG/TIFF/WEBP и не читает видео).
Место на диске под индекс (SQLite) и миниатюры растёт с размером коллекции;
сортировка с `--copy` требует примерно ×2 размера коллекции, `--link`
(жёсткие ссылки) — почти ничего.

Цифры выше — с нашего железа, не гарантия. Полная раскладка, включая заметки по
RAM/VRAM — в [руководстве пользователя](docs/guide/user-guide.ru.md#2-требования).

## Быстрый старт

```bash
# Установка один раз — выберите профиль под своё железо
uv tool install ".[cpu]"                # без NVIDIA GPU — кладёт `sorta` на PATH
# или
uv tool install ".[gpu]"                # NVIDIA GPU + драйвер CUDA 13
cp config.example.yaml config.yaml      # задайте `sources` и `language`
# exiftool обязателен для HEIC/RAW/видео — установите его сначала (см. «Требования»)

# Проще всего: веб‑приложение
sorta ui                                # http://127.0.0.1:8756 → Обработать папку → просмотр

# Или CLI
sorta index /путь/к/фото                # сканирование
sorta run                               # гео, landmarks, junk + хэши почти-дублей (города+дубли)
sorta run --faces --events              # ...плюс детекция лиц и построение событий
sorta sort --by city --dest /путь/к/разложенному            # dry-run план (CSV + HTML)
sorta sort --by city --dest /путь/к/разложенному --copy --apply   # применить (copy неразрушающий)
sorta undo                              # откатить последний батч при необходимости
```

Разрабатываете код сами? Используйте venv проекта (`uv sync --extra cpu --extra
dev`, активируйте его, дальше те же команды `sorta …` с живыми правками) — оба
set‑once пути, смена профиля и почему голый `uv run sorta …` — не один из них,
см. [«Установка»](docs/guide/user-guide.ru.md#3-установка) в руководстве
пользователя.

Полный обзор (с реальным выводом команд), справочник команд и конфигурации — в
[руководстве пользователя](docs/guide/user-guide.ru.md).

## Безопасность и приватность

- **Оригиналы никогда не изменяются.** Сортировка перемещает/копирует файлы; EXIF
  не переписывается.
- **Dry‑run по умолчанию;** каждая операция журналируется до выполнения;
  `sorta undo` откатывает её.
- **Локально по умолчанию.** Весь ML считается на вашей машине. Онлайн‑провайдеры
  (геокодинг Nominatim: только GPS‑координаты; именование событий через Claude API:
  несколько фото‑образцов события, только если вы включили это явно) выключены по
  умолчанию — что именно отправляет каждый из них, см. [SECURITY.md](SECURITY.md).
- **Документы** (паспорта, чеки, медицинские бумаги…) собираются в локальную
  review‑папку `_Документы/` и обрабатываются только на вашей машине.
- Веб‑приложение слушает только `127.0.0.1`.

Подробности — в [SECURITY.md](SECURITY.md).
